S3 Explorer — Asset Generator

Generate the social and PWA images for the website. Click each Download button, then move the file to the listed path in the codebase.

og:image (default) public/og-image.png 1200×630

Optional. Per-page override for /buy — wire it via useSeoMeta({ ogImage: ... }).

App icons (square)

4. Apple touch icon

iOS / Safari public/apple-touch-icon.png 180×180

Standard size when added to iOS home screen. Replace icon-256.png in nuxt.config.ts link.

5. Android Chrome 192

PWA public/android-chrome-192x192.png 192×192

PWA "any" purpose icon. Wire it in public/manifest.json.

6. Android Chrome 512

PWA public/android-chrome-512x512.png 512×512

PWA splash / large icon. Slightly upscaled from 256×256 source — acceptable for PWA usage.

Where to save them, and what to update

File Save to Then update
og-image.png public/og-image.png Already referenced in nuxt.config.ts meta — no change needed once file exists.
og-image-buy.png public/og-image-buy.png In app/pages/buy.vue add ogImage: 'https://s3explorer.accessdevops.com/og-image-buy.png' to useSeoMeta.
og-image-news.png public/og-image-news.png Same pattern in app/pages/news.vue.
apple-touch-icon.png public/apple-touch-icon.png In nuxt.config.ts: change { rel: 'apple-touch-icon', sizes: '256x256', href: '/icon-256.png' } to { rel: 'apple-touch-icon', sizes: '180x180', href: '/apple-touch-icon.png' }.
android-chrome-192x192.png public/android-chrome-192x192.png In public/manifest.json replace the existing icon-128.png entry with the 192×192 file (same purpose).
android-chrome-512x512.png public/android-chrome-512x512.png In public/manifest.json replace the existing icon-256.png entry with the 512×512 file.
